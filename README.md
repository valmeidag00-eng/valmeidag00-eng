Proyectos:
Proyecto 1 Diseño de checklists básicos para apps web simples (UI, validación de campos, flujo de usuario)
Objetivo: Entender los fundamentos de calidad de software y QA.

Actividades principales:

Conceptos básicos: ¿Qué es QA? Diferencia entre QA y QC.
Tipos de pruebas: funcionales, no funcionales, smoke, regression, exploratorias.
Ciclo de vida del software y metodologías (Waterfall vs Agile/Scrum).
Introducción a bugs: severidad y prioridad.
Logros: Creación de checklists y primeros reportes de defects.

Proyecto 2: Diseño de casos de prueba manuales para funcionalidades web (formularios, navegación).
Objetivo: Aprender a diseñar casos de prueba y usar herramientas de gestión.

Actividades principales:

Técnicas de diseño: clases de equivalencia, valores límite, tablas de decisión.
Formatos de casos de prueba (condiciones previas, pasos, datos de prueba, resultado esperado).
Introducción a Jira para reporte de bugs (creación de tickets, adjuntar capturas).
Pruebas de responsividad y compatibilidad básica.
Logros: +30 casos de prueba, primeros reportes en Jira.

Proyecto 3: Diseño y ejecución de +50 casos de prueba manuales, reporte de 15+ defects en Jira (severidad Media/Alta).
Objetivo: Pruebas manuales completas de una funcionalidad real de app web.

Actividades principales:

Revisión de requisitos funcionales (documento con diseños y descripciones).
Flujos probados: ingresar direcciones ("Desde" y "Hasta"), selección de modo (Óptimo, Flash, Personal), panel de tarifas (Casual, Camping, De lujo), formulario de reserva (licencia, pago), botón "Reservar", ventana de confirmación/cancelación, temporizador.
Pruebas de responsividad (800x600 y 1920x1080).
Validación de UI (íconos de autos, mapa, mensajes de error).
Logros: Cobertura completa de flujo, identificación de defects críticos, reportes detallados.

Proyecto 4: Diseño y ejecución de 20+ casos de prueba, reporte de 10+ defects en Jira (severidad Media/Alta).
Objetivo: Pruebas de API REST con Postman y diseño de casos avanzados.

Actividades principales:

Endpoint 1: POST /kits/:id/products – agregar productos a kits.
Validación de límites (máximo 30 productos únicos).
Negativos: IDs inválidos ("abc", 0, -1, nulo, decimal), quantity inválido (negativo, texto, decimal, vacío), body mal formado.
Técnicas: clases de equivalencia, valores límite, pruebas de robustez.
Logros: Validación completa de APIs, reporte detallado con bodies JSON y capturas.
